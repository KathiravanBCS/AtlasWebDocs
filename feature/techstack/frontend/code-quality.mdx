## Linting — ESLint

| Technology | Description | Study link |
| ---------- | ----------- | ---------- |
| **ESLint** | Pluggable linter that catches bugs and enforces conventions. | [eslint.org/docs/latest](https://eslint.org/docs/latest/) |
| **ESLint React** | React-specific lint rules. | [github.com/jsx-eslint/eslint-plugin-react](https://github.com/jsx-eslint/eslint-plugin-react) |
| **React Refresh** | Lint rules ensuring components are Fast Refresh-safe. | [github.com/ArnaudBarre/eslint-plugin-react-refresh](https://github.com/ArnaudBarre/eslint-plugin-react-refresh) |
| **JSX A11Y** | Accessibility lint rules for JSX. | [github.com/jsx-eslint/eslint-plugin-jsx-a11y](https://github.com/jsx-eslint/eslint-plugin-jsx-a11y) |
| **TypeScript ESLint** | Lint rules and parser for TypeScript. | [typescript-eslint.io](https://typescript-eslint.io) |
| **ESLint Mantine** | Mantine's recommended ESLint configuration. | [github.com/mantinedev/eslint-config-mantine](https://github.com/mantinedev/eslint-config-mantine) |

## Formatting — Prettier

| Technology | Description | Study link |
| ---------- | ----------- | ---------- |
| **Prettier** | Opinionated code formatter that ends style debates. | [prettier.io/docs](https://prettier.io/docs/) |
| **Prettier Sort Imports Plugin** | Automatically sorts and groups import statements. | [github.com/trivago/prettier-plugin-sort-imports](https://github.com/trivago/prettier-plugin-sort-imports) |

<Note>
  These tools enforce the rules in the Playbook's
  [Development Standards](/feature/playbook/development-standards) — standards that can
  be a lint rule should be.
</Note>
