A clean Mintlify repo is mostly content plus one config file.

```text
docs/
├── docs.json            # The single source of configuration
├── index.mdx            # Landing page
├── favicon.svg
├── logo/
│   ├── light.svg
│   └── dark.svg
├── images/              # Screenshots, diagrams
├── snippets/            # Reusable MDX fragments
├── essentials/          # Hand-written guides (any folder name)
├── api-reference/       # API docs + OpenAPI/AsyncAPI specs
│   └── openapi.json
└── llms.txt             # (optional) — auto-generated if omitted
```

<Note>
  Folder names are conventions, not requirements. Mintlify resolves pages by the
  **path you list in `docs.json`**, not by directory magic. A page at
  `guides/auth.mdx` is referenced as `"guides/auth"` (no extension).
</Note>
